# 南华期货策略 · 可机器验收的回测规格（SSOT）

> 用途：① 作为 **OpenHands vs Stagent 对比**的统一输入需求 + 中立验收；② 作为 Stagent **T4 一直缺的机读策略契约**。
> 原始需求是自然语言 + 看盘术语，**大量名词未定义**。本规格对每个模糊点做出**明确决策**并标注 `【待确认】`——你可逐条改，但在确认前以本文为准。
> **范围铁律**：只做**回测 / 模拟**，不接实盘券商、不自动下真实订单（安全 + 可验证）。

---

## 0. 范围与非目标

- **范围**：给定历史 K 线数据，离线计算指标 → 判定入场/止损信号 → 输出信号序列 + 回测统计。
- **非目标**：
  - 不证明策略**盈利**（首期只保证"按规格正确计算信号"，不保证赚钱）。
  - 不接实盘下单、不连券商 API、不做实时行情。
  - 不做参数寻优 / 机器学习。
- **验收口径**：产物必须能**独立真实运行**于样本数据并输出**非平凡**信号，且**逐条复现**下文的 oracle 断言——"测试绿但跑不出信号 / 信号恒空"= 空心绿，判不合格。

## 1. 名词与假设决策表（最重要 · 全部 `【待确认】`）

| 原文术语 | 决策（精确定义） | 备注 |
|---|---|---|
| 1 个"点" | 价格的 **1.0 个报价单位**（1 point = 1.0 price unit） | 具体合约 tick 不同，需按南华具体品种确认 |
| 均线（K线指标 5+6+7+8+9+11+20） | 对**收盘价**取 SMA，周期集合 `MA_PERIODS={5,6,7,8,9,11,20}` | 决策用 **SMA**（非 EMA） |
| "5条/前5条线并拢" | 取**前 5 条**均线 `{MA5,MA6,MA7,MA8,MA9}`，其 `max-min < 2.0 点` | 用前 5 条，不含 11/20 |
| "穿插20日线" | 收盘价穿越 `MA20`：做空=下穿（`close[t-1]≥MA20[t-1] 且 close[t]<MA20[t]`），做多=上穿 | |
| BOLL 20+2 | `mid=SMA20(close)`，`up=mid+2σ`，`low=mid-2σ`，σ=最近20根收盘的**总体**标准差 | |
| "布林带横盘振荡不做" | 过滤器：`带宽% = (up-low)/mid*100`，若 `带宽% < 1.5` 视为横盘 → **跳过不交易** | 阈值 1.5% 为决策值 |
| VOL 3+100 | 成交量均线 `MAVOL3`、`MAVOL100` | |
| "白线" | = `MAVOL3`（较快线）；"白线上升"=`MAVOL3[t]>MAVOL3[t-1]` | 决策白线=快线 |
| "红柱/绿柱" | 量柱按当根 K 线涨跌着色：`绿柱`=`close<open`（跌），`红柱`=`close≥open`（涨） | |
| "倍量" | `volume[t] ≥ 2 × volume[t-1]` | 决策对比前一根；备选=2×MAVOL3 |
| MACD 14+53+60 | `DIF=EMA14(close)-EMA53(close)`，`DEA=EMA60(DIF)`，`hist=2×(DIF-DEA)` | 参数非标准，按原文直取 |
| "黄白线"、"靠近零轴" | 白线=`DIF`，黄线=`DEA`；靠近零轴=`|DIF|<ε 且 |DEA|<ε`，`ε=0.5 点` | ε 为决策值 |
| "绿柱加长 / 红柱加长" | 绿柱加长=`hist<0 且 hist[t]<hist[t-1]`；红柱加长=`hist>0 且 hist[t]>hist[t-1]` | |
| CCI 89 | `CCI=(TP-SMA89(TP))/(0.015×平均绝对偏差)`，`TP=(H+L+C)/3` | |
| "二次下穿/上穿0轴" | 在 `半小时` 窗口内统计 CCI 与 0 轴的穿越次数，**第 2 次**穿越当根触发；下穿=`CCI[t-1]≥0 且 CCI[t]<0` | 窗口=30 分钟（3min→10根，1min→30根） |
| "上证指数/深证成指共振" | 同一时间戳，两个指数 `close` 同时位于各自 `MA20` 的**下方**（做空）或**上方**（做多） | 指数也需提供数据 |
| "白点"（仅多买入点出现） | **`【高度待确认】`** 看盘软件的可视标记，原文无定义。**临时决策**：将"白点"定义为"该周期其余多头条件全部成立"的合成确认标记（即软件渲染的"信号确认"），不引入额外独立条件 | 必须向需求方确认其真实含义 |
| "空买入 / 多买入" | 决策：`空买入`=**开空仓**入场点；`多买入`=**开多仓**入场点 | |

## 2. 数据契约（输入）

> 多周期 + 双指数。所有 CSV UTF-8、首行表头、时间升序。

**主力合约 K 线**（1 分钟、3 分钟各一份）：`bars_1m.csv` / `bars_3m.csv`

| 列 | 类型 | 说明 |
|---|---|---|
| `timestamp` | ISO8601 | 该 K 线**收盘**时间 |
| `open` `high` `low` `close` | float | 价格 |
| `volume` | float | 成交量 |

**指数行情**（与主力同周期，至少含判定所需周期）：`index_sh.csv`（上证）、`index_sz.csv`（深证成指）

| 列 | 类型 |
|---|---|
| `timestamp` | ISO8601 |
| `close` | float |

> 约束：1m 与 3m、指数与主力**时间对齐**（按 `timestamp`）；缺口需前向填充或显式标记，不得静默丢弃。

## 3. 指标定义（精确公式，按 §1 决策）

```
SMA_n(x)[t]        = mean(x[t-n+1 .. t])
EMA_n(x)[t]        = α·x[t] + (1-α)·EMA_n[t-1],  α = 2/(n+1)
MA{p}              = SMA_p(close)            for p in {5,6,7,8,9,11,20}
BOLL: mid=SMA20(close); σ=stdev_pop(close[t-19..t]); up=mid+2σ; low=mid-2σ
带宽%              = (up-low)/mid × 100
MAVOL3/MAVOL100    = SMA3(volume)/SMA100(volume)
DIF=EMA14(close)-EMA53(close); DEA=EMA60(DIF); hist=2×(DIF-DEA)
TP=(high+low+close)/3
CCI=(TP - SMA89(TP)) / (0.015 × mean(|TP[i]-SMA89(TP)[t]|, i=t-88..t))
```

> 暖机：所有指标在样本前 **≥100 根**之前不产生信号（最长周期 MAVOL100），不足则该根信号判定为 `false`。

## 4. 入场信号（可计算谓词）

### 4.1 空买入点（开空），以下条件**全部**为真（在 3 分钟主周期判定）

| # | 条件 | 谓词 |
|---|---|---|
| C1 | 前5均线并拢后下穿20线 | `spread({MA5..MA9}) < 2.0`（近 K 根内成立过）`且` `close 下穿 MA20` |
| C2 | 非横盘 | `带宽% ≥ 1.5`（否则不做） |
| C3 | VOL 白线上升 + 绿柱倍量 | `MAVOL3↑` `且` `close<open` `且` `volume[t] ≥ 2×volume[t-1]` |
| C4 | MACD 黄白线靠近零轴 + 绿柱加长 | `|DIF|<0.5 且 |DEA|<0.5` `且` `hist<0 且 hist[t]<hist[t-1]` |
| C5 | CCI 二次下穿0轴 | 半小时窗口内第 2 次 `CCI 下穿 0` |
| C6 | 1分钟同向下穿20线 | `1m: close 下穿 MA20` |
| C7 | 双指数共振（下方） | `index_sh.close<MA20_sh 且 index_sz.close<MA20_sz` |

`空买入信号[t] = C1 ∧ C2 ∧ C3 ∧ C4 ∧ C5 ∧ C6 ∧ C7`

> `【待确认】`C1 的"并拢后穿插"两动作的时间窗 K：决策 `K=3` 根 3 分钟内先并拢、随后下穿。

### 4.2 多买入点（开多），镜像 4.1

| # | 条件 | 谓词 |
|---|---|---|
| C1 | 前5均线并拢后上穿20线 | `spread({MA5..MA9}) < 2.0` `且` `close 上穿 MA20` |
| C2 | 非横盘 | `带宽% ≥ 1.5` |
| C3 | VOL 白线上升 + 红柱倍量 | `MAVOL3↑ 且 close≥open 且 volume[t] ≥ 2×volume[t-1]` |
| C4 | MACD 零轴之上靠近0轴 + 红柱加长 | `0<DIF<0.5 且 0<DEA<0.5（或 DIF,DEA≥0 且靠近）` `且` `hist>0 且 hist[t]>hist[t-1]` |
| C5 | CCI 半小时内二次上穿0轴 | 半小时窗口内第 2 次 `CCI 上穿 0` |
| C6 | 1分钟也上穿20线 | `1m: close 上穿 MA20` |
| C7 | 1m 与 3m 均带"白点" | 见 §1 决策（= 其余多头条件确认）`【待确认】` |
| C8 | 双指数共振（上方） | `index_sh.close>MA20_sh 且 index_sz.close>MA20_sz` |

`多买入信号[t] = C1 ∧ … ∧ C8`

## 5. 止损与离场规则

> `【待确认】`原文止损含"昨天对冲单 / 当天开单"两种情形，措辞歧义大。下表**当天开单**部分判定清晰、优先实现；**昨天对冲单**部分按最合理解读编码并强标待确认。

### 5.1 当天开单（清晰，优先实现）

| 开仓方向 | 止损点 | 触发动作 | 持仓 |
|---|---|---|---|
| 多单 | 开仓价 **− 15 点** | 触及 → **开空单对冲**，放到明天 | 一般次日平仓 |
| 空单 | 开仓价 **+ 15 点** | 触及 → **开多单对冲**，放到明天 | 一般次日平仓 |

### 5.2 昨天对冲单的平仓单（歧义，标待确认）

| 平仓单方向 | 止损 | 动作 |
|---|---|---|
| 多单 | 买入点位 **+15** 设止损 | 平空单 + 开空单对冲 `【待确认】` |
| 空单 | 买入点位 **−15** 挂单 | 平多单 + 反向平仓保护 `【待确认】` |

### 5.3 离场

- 默认 **持有至次一交易日平仓**（`【待确认】`具体平仓时刻，决策=次日开盘第 1 根）。
- 触及止损则按 §5.1/§5.2 执行对冲。

## 7. 验收（中立 oracle · 三层，machine-checkable）

> 验收实现须**与被测工具无关**（不用 Stagent 自带门当裁判），两边同一把尺子。

**L1 指标单元 oracle**（手算可核）：给极小 fixture（如 6 个收盘价 `[10,10,10,10,10,10]`）→ `SMA5=10`、`spread=0`；`[10,11,12,13,14]` 前5均线 → 给出期望 `spread`。每个指标≥1 条手算断言。

**L2 谓词 oracle**：构造命中/不命中各 1 条 fixture，断言 `C1..C8` 布尔值与 §4 一致（尤其 `二次穿0轴`、`倍量`、`横盘过滤`、`共振` 各单独测）。

**L3 端到端**：给一段 `bars_3m/1m/index_*` 样本 CSV + `expected_signals.csv`（golden）。任一正确实现须复现 golden（时间戳完全匹配、`signal` 一致）。
- `【产出注记】`golden 需先用一份**参考实现**生成并人工抽检（这是 SSOT 的一部分工作，本规格只定义格式与方法）。

**空心绿检测**（套到两个工具产物）：
1. 真实跑：`python main.py --bars-3m … --bars-1m … --index-sh … --index-sz …`，断言 `signals.csv` 行数 > 0 且 `summary` 非全 0；
2. 复现 L1/L2 oracle；
3. 测试不得把指标/数据整体 mock（协作者 mock 假绿）；
4. 用"全部条件应命中的构造样本"喂入，必须产出 `OPEN_*`——否则恒空判空心绿。

## 8. 用于 OpenHands vs Stagent 对比的注记

- **同模型**：两边都用 DeepSeek。
- **同输入**：把本文 §0–§6 作为需求喂给两个工具（§7 留作裁判，不喂）。
- **各跑 N≥3**，报成功率（按 §7 验收，不看"看起来做完了"）。
- **诚实标注**：本任务属"领域语义重 + 不可约推理"，Stagent 自评其卡点在**模型能力**非引擎；故对比看点是 **谁产出的空心绿更少 / 谁更需人工兜底 / 谁更老实地暴露未定义项（§1 那些待确认）**，而非"谁策略对"。
- **公平性披露**：Stagent 在本领域（T4）迭代过多轮，非冷启动；若与其已有 T4 产物对比须注明。
